// Express.js REST API Example const express = require('express'); const app = express(); // Middleware app.use(express.json()); // Sample database const users = [ { id: 1, name: 'John Doe', email: 'john@example.com' }, { id: 2, name: 'Jane Smith', email: 'jane@example.com' } ]; // GET /api/users - Retrieve all users app.get('/api/users', (req, res) => { res.json({ success: true, count: users.length, data: users }); }); // POST /api/users - Create new user app.post('/api/users', (req, res) => { const newUser = { id: users.length + 1, ...req.body }; users.push(newUser); res.status(201).json({ success: true, data: newUser }); }); // GET /api/users/:id - Get user by ID app.get('/api/users/:id', (req, res) => { const user = users.find(u => u.id === parseInt(req.params.id)); if (!user) { return res.status(404).json({ success: false, error: 'User not found' }); } res.json({ success: true, data: user }); }); // Start server const PORT = 3000; app.listen(PORT, () => { console.log(`Server running on port ${PORT}`); });

Design Patterns Visualization

Factory Method Pattern

Creates objects without specifying the exact class to instantiate. Promotes loose coupling and makes systems easier to extend.

Use Case: When you need to create objects based on configuration or runtime conditions.

Adapter Pattern

Allows incompatible interfaces to work together. Acts as a bridge between two systems without changing their existing code.

Use Case: When integrating with third-party libraries or legacy systems.

Observer Pattern

Defines a one-to-many relationship where multiple objects are notified automatically when another object's state changes.

Use Case: Event-driven systems, publish-subscribe mechanisms.

Interview Preparation

Practice with 15 essential back-end development interview questions

Question 1 Easy
What is the difference between REST and GraphQL APIs?

Answer:

REST: Uses multiple endpoints, fixed data structure, standard HTTP methods, built-in caching, simpler implementation.

GraphQL: Single endpoint, flexible queries, client specifies needed data, reduces over/under-fetching, better for complex relationships.

Choose REST when: Simple CRUD operations, caching is important, working with HTTP proxies.

Choose GraphQL when: Multiple clients with different data needs, complex data relationships, need for real-time updates.

Question 2 Medium
Explain database indexing and how it improves query performance.

Answer:

Database Indexing: Creates a data structure (usually B-tree) that allows faster data retrieval without scanning the entire table.

Performance Benefits:

  • Speeds up SELECT queries dramatically
  • Improves JOIN performance
  • Enforces uniqueness with unique indexes
  • Optimizes ORDER BY and GROUP BY operations

Trade-offs: Slows down INSERT/UPDATE/DELETE operations, uses additional disk space, requires maintenance.

Question 3 Hard
How does JWT authentication work and what are its security considerations?

Answer:

JWT Structure: Header.Payload.Signature

Authentication Flow:

  1. Client sends credentials to authentication endpoint
  2. Server validates credentials and creates JWT
  3. JWT is signed with server secret and sent to client
  4. Client includes JWT in Authorization header for subsequent requests
  5. Server verifies JWT signature and processes request

Security Considerations:

  • Store tokens securely (HttpOnly cookies for web)
  • Use short expiration times
  • Implement token blacklisting for logout
  • Always use HTTPS
  • Validate token signature properly
Question 4 Medium
What are the SOLID principles in Object-Oriented Programming?

Answer:

S - Single Responsibility Principle: A class should have only one reason to change, focusing on a single responsibility.

O - Open/Closed Principle: Software entities should be open for extension but closed for modification.

L - Liskov Substitution Principle: Objects of a superclass should be replaceable with objects of a subclass without affecting correctness.

I - Interface Segregation Principle: Many client-specific interfaces are better than one general-purpose interface.

D - Dependency Inversion Principle: Depend on abstractions, not on concretions.

Question 5 Medium
Explain the difference between SQL and NoSQL databases.

Answer:

SQL Databases:

  • Relational database management systems
  • Structured schema with tables, rows, columns
  • Use SQL for queries
  • ACID compliance (Atomicity, Consistency, Isolation, Durability)
  • Vertical scaling
  • Examples: PostgreSQL, MySQL, Oracle

NoSQL Databases:

  • Non-relational databases
  • Flexible schema or schema-less
  • Various data models (document, key-value, graph, column-family)
  • BASE model (Basically Available, Soft state, Eventual consistency)
  • Horizontal scaling
  • Examples: MongoDB, Cassandra, Redis
Question 6 Hard
What is database connection pooling and why is it important?

Answer:

Connection Pooling: A cache of database connections maintained so that the connections can be reused when future requests to the database are required.

How It Works:

  1. Application starts with a pool of database connections
  2. When needed, a connection is checked out from the pool
  3. After use, the connection is returned to the pool
  4. Connections are reused instead of created/destroyed

Benefits:

  • Reduces connection establishment overhead
  • Improves application performance
  • Manages database connection limits efficiently
  • Provides better resource utilization
  • Prevents connection leaks with proper management
